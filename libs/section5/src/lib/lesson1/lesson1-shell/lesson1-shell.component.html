This first lesson consists of creating our base building blocks for our wizard - creating a boolean, number, and string-based form (for each of the config settings that our app is managing).

Let's start with the boolean form. We'll use the mat-checkbox provided by @angular/material, and allow for a `storeValue` input that we'll use to display an indicator in the case where a change exists.

The template is already provided for you, all that remains to be done is implement the main methods required by the ControlValueAccessor. 

form value: {{ booleanControl.value }}

Next for the number form, we'll do a very similar task to the boolean form, however rather than mat-checkbox, we'll be wrapping a mat-form-field. 

form value: {{ numberControl.value }}
form errors: {{ numberControl.errors | json }}
form value: {{ numberControl.value }}
form errors: {{ numberControl.errors | json }}

Finally for the string form, we'll go through this pattern again. 

form value: {{ stringControl.value }}
form errors: {{ stringControl.errors | json }}
form value: {{ stringControl.value }}
form errors: {{ stringControl.errors | json }}
