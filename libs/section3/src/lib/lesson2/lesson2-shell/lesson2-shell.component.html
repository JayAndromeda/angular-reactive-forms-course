Another common use-case for the `ControlValueAccessor` is to encapsulate a FormGroup and instead expose a FormControl that will manage a single object (sometimes changing the format/shape of that object).

Let's see how that plays out in this example, where we'll wrap a FormGroup that wraps two FormControls - one using a HTML date input element and the other using an HTML time input element - and allow a user to interact with a single FormControl that takes a JS `Date` object as it's value.
